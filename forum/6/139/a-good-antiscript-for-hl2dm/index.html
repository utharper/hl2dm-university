Keeper

2007-05-07 21:27:39

All the commands get processed on the client and then get sent on to the server. So the server might not even see some of the commands that can be found in the "alias" scripts.

I asked Valve about the alias and why it was necessary to have in DM, they responded:
Aliases are used by the game code itself so we can't just turn it off.
If somebody does have an anti-script out there, they probably have some kind of time tolerance on executing commands. If they get a block of commands that hit at the same time, then they probably allow the first one through and ignore the rest. "Wait" isn't even seen by the server, so they would have to be doing some pretty serious hacking to get to this information (if it's even possible).

I get the feeling that if pushed, valve could disable the alias command for clients, and still let their internal code run the alias commands. But I'm not holding my breath on this one.
