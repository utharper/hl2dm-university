Deathwish wrote:
Walking Target wrote:I knew someone would say that.

I prefer to have all my settings in a read-only autoexec file, and allow the game to write to the config.
wtf?? whats the point of a read only autoexec cfg file lol
config.cfg and autoexec.cfg are both common file names and may be targeted by malicious server admins. Either one can get slow hacked. Having an autoexec.cfg that is set to read only allows you to separate out your custom settings from the file that the game writes to, and setting it to read only protects it from slow hacking. If your config ever gets corrupted, you can simply delete it and let the game create a new one, then exec your autoexec and you are good to go again. It allows the best of both worlds, a config that can be changed on the fly, and a read only file that keeps your standard settings.
