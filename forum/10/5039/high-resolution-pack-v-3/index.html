About:

This pack scales up most all textures in the game to HD-quality and gives it a more realistic feel. Depending on the options you choose, this pack will provide HD player models, HD surroundings, HD weapons, and HD Props. This pack does all this without lowering your framerate.
Please note - this pack will work on every server except those set to sv_pure 1 or 2. If this pack isn't working for you in-game, its because the server you're on is not sv_pure 0.

What's Included in this Pack Exactly?

Hi-Resolution Explosions
Hi Resolution Blood Decals
Hi Resolution Medkits
Hi Resolution Suit Chargers
Hi Resolution Health Chargers
Hi Resolution Weapons
Hi Resolution Weapon Models
Hi Resolution Barrels, Crates, Cabinets, Radiators, etc...
Hi Resolution Players
Hi Resolution Hands
Hi Resolution Textures for Maps
Hi Resolution Gravity Gun
Fixes USP no-shell bug
Adds a really neat blue effect with orbs.

Screenshots: Download Screenshot Pack HERE.

This pack comes in four parts, so you can pick and choose exactly what you want to install.

Weapons (30mb) - HERE

Players (145mb) - HERE

Textures (200mb) - HERE

Props (200mb) - HERE

Installing: Put the "materials" folder and/or "models" folders into C:/Program Files/Steam/SteamApps/*Your Steam Name*/Half-Life 2:Deathmatch/hl2mp/

Uninstalling: Just delete those two folders.

Credits: I did not create any of these textures or skins. I merely collected them from the broader modding community and put them all together.

I have a gateway that is similar. not GFX series though. Came with AMD 64x2 6000+ @ 3.1ghz with Onboard video. 6150se Nforce 430, same as above.

I got 15-30 FPS with default settings in game for that "card".

6 series should handle 30-80 fps I would imagine, not much more than that, not sure, never used a 6 series dedicated GPU.

reason that 6150 is so crap, is the simple fact it's onboard video. It uses a shared video system memory setup, meaning it uses your system ram, setting aside x amnt (configurable in BIOS) for rendering images. That takes up CPU. It also doesn't have a dedicated GPU (or mulitple GPU cores) doing all the rendering. It works in tandem with that little onboard GPU and your CPU (does most of the work in onboard video card scenarios).

In effect, your losing out CPU power * 2 - due to "shared system memory" and the CPU time being shared heavily from running a game/ Operating system / graphics.
Thus the crappy frame-rate. FPS = CPU power + dedicated GPU power. CPU accounts for a lot of FPS; if it's busy, it will be crap.
